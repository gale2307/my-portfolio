# Phase 1 ‚Äî UI Design Contract

> Visual and interaction contract for Phase 1: Foundation + Hero.
> Generated by gsd-ui-phase, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | react-icons/si (Simple Icons) |
| Font (body) | Geist ‚Äî via `next/font/google` |
| Font (accent/mono) | Geist Mono ‚Äî via `next/font/google` |

**No shadcn, no Radix, no UI kit.** All components are hand-rolled Tailwind. This is an intentional aesthetic decision ‚Äî the "nonchalant" tone requires full control over every visual element.

## Color Palette

| Role | Hex | Tailwind Token | Usage |
|------|-----|---------------|-------|
| Background (60%) | `#1a1a1a` | `--color-bg` | `<html>` background ‚Äî set in `globals.css` before JS loads |
| Surface (30%) | `#242424` | `--color-surface` | Nav backdrop, subtle card tint |
| Primary text | `#f5f5f5` | `--color-text` | All headings, body copy |
| Secondary text | `#a3a3a3` | `--color-text-muted` | Section labels, meta text, captions |
| Accent (10%) | `#c8964a` | `--color-accent` | CTA button border, hover states, link underlines |
| Destructive | n/a | n/a | Not in Phase 1 scope |

**Accent reserved for:** CTA button (border + text), hover state on nav links, link hover underlines. Nothing else. The accent should whisper ‚Äî if it reads as yellow, it's too bright.

**No other colors.** Background, off-white text, one muted amber accent ‚Äî nothing else per D-10.

**Dark mode flash prevention:** `background-color: #1a1a1a` set on `html` element in `globals.css` as a plain CSS rule (not inside `@layer` or Tailwind utilities). This applies before any JavaScript loads, preventing FOUC.

## Typography

All sizes in `globals.css` via `@theme` block (Tailwind v4 CSS-first config).

| Role | Element | Size | Weight | Line Height | Font | Tracking |
|------|---------|------|--------|-------------|------|----------|
| Display | Hero name | `clamp(48px, 6vw + 8px, 72px)` | 700 (bold) | 1.1 | Geist | `tracking-tight` |
| Heading | Section titles | `36px` | 600 (semibold) | 1.2 | Geist | normal |
| Subheading | Role descriptor | `18px` | 300 (light) | 1.4 | Geist | `tracking-wide` |
| Body | Tagline, prose | `16px` | 300 (light) | 1.7 | Geist | normal |
| Label (mono) | Section labels | `12px` | 400 (normal) | 1.5 | Geist Mono | `tracking-widest` |
| Nav | Nav links | `14px` | 500 (medium) | 1.5 | Geist | normal |
| CTA | Button text | `15px` | 500 (medium) | 1 | Geist | `tracking-wide` |

**Font weight strategy (D-05):** High contrast between structural and body elements. Bold/semibold for names and headings only. Light/normal for everything else. This creates visual hierarchy without relying on size alone.

**Mono label convention (D-03):** Section labels rendered with Geist Mono, `#a3a3a3`, `12px`, `tracking-widest`, uppercase. Format: `// section-name` (e.g., `// about`, `// work`). This is a distinctive typographic detail ‚Äî applied consistently from Phase 1 onward.

## Spacing Scale

All spacing from a strict 4px grid.

| Token | Value | Usage |
|-------|-------|-------|
| xs | `4px` | Icon gaps, tight inline spacing |
| sm | `8px` | Button padding (vertical), compact spacing |
| md | `16px` | Default element gap, button horizontal padding |
| lg | `24px` | Within-section spacing |
| xl | `32px` | Between hero content elements |
| 2xl | `48px` | Hero bottom padding |
| 3xl | `64px` | Nav height, major section breaks |

**Container:** `max-w-5xl` (`1024px`) with `px-6` (`24px`) side padding. Centered via `mx-auto`. This keeps content readable and left-aligned without stretching to full-bleed on large screens.

**Hero section height (D-12):** `min-h-[70vh]` with `max-h-[80vh]` effectively via `h-[75vh]`. Top of next section peeks below the fold ‚Äî invites scroll.

Exceptions: none.

## Layout Contracts

### Navigation Bar

```
[ Name/Logo (optional) ]          [ About  Skills  Work  Contact ]
```

- **Position:** `fixed top-0 left-0 right-0 z-50`
- **Height:** `64px` (3xl token)
- **Background:** `bg-[#1a1a1a]/80 backdrop-blur-md` ‚Äî glassmorphism effect
- **Border:** `border-b border-white/5` ‚Äî barely perceptible separator
- **Container:** Same `max-w-5xl mx-auto px-6` as body content
- **Links:** `About`, `Skills`, `Work`, `Contact` ‚Äî smooth-scroll to section anchors
- **Link style:** `#f5f5f5` default, `#c8964a` on hover (accent), no underline by default
- **Logo/name area:** Left side, user's first name or initials in Geist 14px medium ‚Äî Claude's discretion on whether to include; if included, links back to `#top`
- **Mobile:** Links collapse or remain inline ‚Äî Phase 1 scope is desktop-first; responsive is Phase 4 polish

### Hero Section

```
[padding-top: nav height = 64px]

  {// freelance}                  ‚Üê Geist Mono label, muted, 12px

  {Full Name}                     ‚Üê Display, 64px, bold, #f5f5f5
  {Freelance Software Engineer}   ‚Üê Subheading, 18px, light, #a3a3a3
  {One-line tagline}              ‚Üê Body, 16px, light, #f5f5f5, max 80 chars

  [ Get in touch ]                ‚Üê CTA button

```

- **Section:** `h-[75vh] flex flex-col justify-center` ‚Äî vertically centered
- **Content:** Left-aligned (`text-left`), max-width `max-w-2xl` to constrain line length
- **Stagger:** Each element animates in sequence (see Animation Contract below)
- **Gap between name and role:** `8px` (sm)
- **Gap between role and tagline:** `16px` (md)
- **Gap between tagline and CTA:** `32px` (xl)

## CTA Button Contract

- **Label:** "Get in touch" (exact copy ‚Äî no variation)
- **Behavior:** Smooth-scroll to `#contact` section
- **Style:** Ghost/outline ‚Äî `border border-[#c8964a] text-[#c8964a]` ‚Äî transparent background
- **Hover:** `bg-[#c8964a]/10` ‚Äî very subtle amber fill, no background flash
- **Padding:** `px-6 py-3` (`24px` horizontal, `12px` vertical)
- **Border radius:** `rounded-sm` ‚Äî subtle, not pill-shaped; maintains the nonchalant tone
- **Font:** 15px, `font-medium`, `tracking-wide`, Geist
- **Transition:** `transition-colors duration-200`

## Animation Contract

**Hero entrance animation (D-15):**

All elements use Framer Motion with `initial={{ opacity: 0, y: 16 }}` ‚Üí `animate={{ opacity: 1, y: 0 }}`. Stagger delay applied per element. Plays once on mount ‚Äî not a scroll-triggered animation.

| Element | Delay | Duration |
|---------|-------|----------|
| Mono label (`// freelance`) | 0ms | 400ms |
| Hero name | 100ms | 500ms |
| Role descriptor | 200ms | 500ms |
| Tagline | 300ms | 500ms |
| CTA button | 450ms | 400ms |

- **Easing:** `easeOut` on all elements
- **Drift:** `y: 16` ‚Üí `y: 0` ‚Äî subtle, not dramatic. 16px is the ceiling; go lower if it feels heavy during implementation.
- **`prefers-reduced-motion`:** When enabled, skip animation (set `initial={{ opacity: 1, y: 0 }}` immediately). Detected via `useReducedMotion()` from Framer Motion.
- **`once: true`:** Not applicable ‚Äî this is a mount animation, not `whileInView`. The animation plays once and stays.

**FadeIn wrapper (for nav):**

The `<FadeIn>` wrapper component (`components/ui/FadeIn.tsx`) is established in Phase 1 for reuse by all subsequent phases. It uses `whileInView` + `viewport={{ once: true }}`. The Hero does NOT use `FadeIn` ‚Äî it uses a direct mount animation since it's always above the fold.

## Tailwind v4 Token Configuration

Declare in `globals.css` under `@theme`:

```css
@theme {
  --color-bg: #1a1a1a;
  --color-surface: #242424;
  --color-text: #f5f5f5;
  --color-text-muted: #a3a3a3;
  --color-accent: #c8964a;
  --font-sans: var(--font-geist);
  --font-mono: var(--font-geist-mono);
}
```

CSS custom properties for runtime use (outside `@theme`):

```css
:root {
  background-color: #1a1a1a; /* flash prevention ‚Äî must be plain CSS, not Tailwind */
  color: #f5f5f5;
}
```
