# Wave 3 — Vercel Deployment + Custom Domain

**Goal:** Push the project to GitHub, connect it to Vercel, trigger a successful production build, and configure the custom domain with DNS records so the site is accessible via HTTPS on the custom domain.

**Prerequisite:** Wave 1 and Wave 2 complete — `next build` exits cleanly, hero renders with stagger animation locally, all Wave 2 verification criteria pass.

**Requirements covered:** FOUND-04

**Note:** `autonomous: false` — this wave involves human actions (creating a GitHub repo, clicking through the Vercel dashboard, updating DNS at a registrar). The tasks below are step-by-step instructions for the engineer to follow.

  </read_first>
  <action>

    In the project configuration screen:
    - Framework Preset: Next.js (should be auto-detected)
    - Root Directory: `.` (default — project root)

    If the Vercel build fails: check the build log for TypeScript or ESLint errors. These errors must not exist locally (`next build` passes locally per Wave 1/2 prerequisites). Fix locally, push, and let Vercel auto-redeploy.

<task id="T15" title="Configure custom domain in Vercel and update DNS at registrar">
  <read_first>
    - .planning/REQUIREMENTS.md — FOUND-04: "accessible on a custom domain"
  </read_first>
  <action>
    In the Vercel project dashboard, navigate to: Settings → Domains → "Add Domain".
    Enter the custom domain (e.g., `nicholaswillian.com`). Click "Add".

    Vercel displays one of two DNS configuration options — choose based on your registrar setup:

    Option A — Vercel nameservers (recommended if you can change nameservers at your registrar):
    Update the domain's nameservers to: `ns1.vercel-dns.com` and `ns2.vercel-dns.com`
    Vercel will manage all DNS records automatically including SSL provisioning.

    Option B — External DNS (if keeping your registrar's DNS):
    Add an A record: `@` → `76.76.21.21` (Vercel's anycast IP)
    Add a CNAME record: `www` → `cname.vercel-dns.com`

    Log into your domain registrar and apply the chosen configuration.

    Return to the Vercel Domains dashboard and wait for status to show "Valid Configuration" (green checkmark). DNS propagation can take 5 minutes to 48 hours depending on TTL.

    Once valid, open `https://{custom-domain}` and verify:
    - Valid SSL certificate (padlock in address bar, no browser security warning)
    - Site loads with dark hero section, identical to the vercel.app URL
    - `http://` requests redirect to `https://` (Vercel enforces this by default — if there is a redirect loop, check for conflicting rules at the registrar)

    If the domain includes `www.`, verify both `nicholaswillian.com` and `www.nicholaswillian.com` resolve correctly (Vercel shows configuration for both).
  </action>
  <acceptance_criteria>
    - Vercel Domains dashboard shows the custom domain with "Valid Configuration" (green checkmark)
    - `https://{custom-domain}` opens with a valid SSL certificate — no browser security warning
    - Site at the custom domain shows the dark hero section with stagger animation
    - `http://` requests redirect to `https://` without a redirect loop
    - Both apex and www subdomain (if applicable) resolve correctly
  </acceptance_criteria>
</task>
