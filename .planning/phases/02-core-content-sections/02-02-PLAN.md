### T5 — Create `lib/data/skills.ts`

```ts
import type { IconType } from 'react-icons';
import { SiTypescript, SiReact, SiNextdotjs, SiNodedotjs, SiPostgresql, SiDocker, SiAmazonwebservices, SiTailwindcss } from 'react-icons/si';

export interface SkillItem {
  name: string;
  icon: IconType;
}

export const skills: SkillItem[] = [
  { name: 'TypeScript', icon: SiTypescript },
  { name: 'React', icon: SiReact },
  { name: 'Next.js', icon: SiNextdotjs },
  { name: 'Node.js', icon: SiNodedotjs },
  { name: 'PostgreSQL', icon: SiPostgresql },
  { name: 'Docker', icon: SiDocker },
  { name: 'AWS', icon: SiAmazonwebservices },
  { name: 'Tailwind CSS', icon: SiTailwindcss },
];
```

**Note:** Importing from react-icons in a data file that's used by a Server Component requires the Skills component itself to be a Client Component (since react-icons renders SVGs). Mark `Skills.tsx` with `'use client'` OR keep the data file as plain name+string pairs and import icons directly in the component. Choose the approach that avoids TypeScript errors cleanest — if `IconType` causes issues, use string icon names and a lookup map.

### T6 — Create `components/sections/Skills.tsx`

Client Component (react-icons requires client-side rendering). Layout:

```
[section id="skills", py-24]
  [max-w-5xl mx-auto px-6]
    <p class="mono label">// skills</p>
    [grid grid-cols-4 gap-8 max-w-2xl]
      [FadeIn delay={index * 0.08} for each skill]
        [flex flex-col items-center gap-2]
          <Icon size={32} color="#a3a3a3" />
          <span class="text-xs font-mono text-[#a3a3a3]">{name}</span>
```

- Grid: `grid-cols-2 sm:grid-cols-4` responsive
- Icon size: 32px, color `#a3a3a3`
- Label: Geist Mono, 12px, `#a3a3a3`
- Stagger: each item gets `delay={index * 0.08}` via FadeIn `delay` prop

**FadeIn delay prop:** Add an optional `delay?: number` prop to `components/ui/FadeIn.tsx` if not already present. Pass it as the `transition.delay` to the motion element.
