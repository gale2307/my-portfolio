# Plan 05-01: Testimonial data model + real client entries + headshot assets

## Objective
Deliver the data-layer slice of the testimonials upgrade: replace the `TestimonialItem` interface (drop `avatarUrl`, add `photo`/`linkedinUrl?`/`source`), repopulate `lib/data/testimonials.ts` with 4–6 real client entries each carrying a `photo` path and a `source`, and create the `/public/images/testimonials/` directory where the headshot JPEGs live. This is a complete vertical slice for the data contract: after this plan, `tsc --noEmit` passes and the data shape consumed by the component (Plan 05-02) is final.
