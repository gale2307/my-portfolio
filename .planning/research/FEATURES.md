# Features Research — Freelance Portfolio

## Section-by-Section: What Works

### Hero

Must-haves:
- Full name, prominently set — the first thing to scan
- One-line tagline describing what you do and for whom (e.g., "I build fast, scalable web products for startups")
- Single primary CTA — "Get in touch" or "Let's work together" linking to the contact section
- Optional: availability status indicator ("Currently available for work" / "Booked until X")

What works in dark typographic hero sections:
- Large display type — the name IS the design element
- Subtle entrance animation (fade + slight upward drift) on load
- High contrast: near-white text on near-black background
- No hero image or avatar — the name and words carry it
- Sparse: hero should breathe, not pack in content

What doesn't work:
- Animated typing effect — feels dated, distracting
- Too many CTAs (hire me + view work + download CV)
- Stock photos or generic background images
- Long introductory paragraph in the hero

### About

Must-haves:
- Who you are and what you do — 2-4 sentences max
- What kinds of problems you solve / who you work with
- Something human — not just a resume summary

What works:
- Short, direct prose — no bullet points
- Hint of personality while staying professional
- No photo required for a dark typographic design (though a high-quality photo can add credibility)

What doesn't work:
- Listing technologies in the About section (that's what Skills is for)
- Third-person writing ("Nicholas is a developer who...")
- Generic phrases: "passionate developer", "loves problem-solving"

### Skills / Tech Stack

Must-haves:
- Visual tech stack icons with labels
- Organized by category (Frontend, Backend, Tools, Cloud/Infra)

What works:
- Icon grid with consistent sizing
- Category grouping helps scanning
- Brief proficiency signal (e.g., "primary" vs "familiar" grouping) without percentages/bars
- No skill meters/progress bars — they communicate nothing meaningful and look unprofessional

What doesn't work:
- Star ratings or percentage bars
- Listing every tool ever touched
- No visual icons (just text lists feel flat)

### Client Logos

Must-haves:
- Consistent logo presentation (same height, grayscale or low-opacity by default)
- Hover state that reveals the company name or color logo

What works:
- Simple horizontal marquee/scroll or grid layout
- Greyscale logos maintain visual coherence in dark theme
- 4-8 logos is the sweet spot — fewer looks thin, more looks cluttered

What doesn't work:
- Mismatched logo sizes/styles
- Color logos with wildly different brand colors competing on a dark background
- "As featured in" headline — implies press coverage, not client work

### Testimonials

Must-haves:
- Quote text
- Name, title, and company of the person
- Optional: small avatar/photo (increases perceived authenticity)

What works:
- Card layout with quote, attribution below
- 2-4 testimonials — enough to establish credibility
- Specific quotes ("reduced our load time by 40%") over vague praise ("great developer")
- Initials avatar if no photo available — better than placeholder icon

What doesn't work:
- First name only (anonymous = untrustworthy)
- More than 5-6 testimonials on a single-page site (padding)
- Star ratings alongside testimonials (feels like a review site)
- Carousel/slider — users won't swipe, and it hides content

### Contact

Must-haves:
- Email link (mailto: or copy-to-clipboard)
- Social links: GitHub + LinkedIn as minimum
- Brief CTA copy framing the section ("Ready to build something?" / "Have a project in mind?")

What works:
- Contact form with name, email, message fields — low friction
- Copy-to-clipboard email button alongside form
- Social icons with accessible labels
- Response time expectation ("I'll get back to you within 24 hours")

What doesn't work:
- Overly long contact forms (phone number, company, budget, timeline fields = friction)
- Just a mailto: link with no form — low commitment asks get ignored
- No social links — feels incomplete for a software engineer

## Table Stakes

- [ ] Name and what you do visible within 3 seconds of landing
- [ ] Single clear CTA in the hero
- [ ] Tech stack displayed visually (icons + labels)
- [ ] At least 2 testimonials with full attribution
- [ ] Working contact method (form or email)
- [ ] Mobile responsive
- [ ] Fast load (< 2s on decent connection)
- [ ] GitHub link
- [ ] LinkedIn link

## Differentiators

- Availability status that updates ("Currently available from September 2025")
- Specific, quantified testimonials ("shipped in 3 weeks", "cut costs by 30%")
- Subtle page entrance animations that don't repeat on scroll
- Clean monospace accent for code-adjacent typography (signals taste)
- Custom 404 page
- Smooth scroll navigation from top nav to sections
- Meta OG image that looks intentional

## Anti-Features

| Feature | Why Skip |
|---------|----------|
| Animated typing effect in hero | Dated, distracting, conflicts with nonchalant tone |
| Skill progress bars / star ratings | Meaningless, look amateur |
| Project case studies | Not the v1 goal; logos + testimonials make the case |
| Blog / writing section | Scope creep for v1 |
| Dark/light mode toggle | Adds complexity, dilutes dark identity |
| Parallax scrolling effects | Overused, can hurt performance and accessibility |
| Chat widget / live chat | Overkill for a personal freelance site |
| PDF resume download | Better to keep them on the site; share LinkedIn instead |
